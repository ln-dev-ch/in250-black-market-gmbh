## Branch strategy

### GitHub Flow - Current Version
Black Market Software is using the GitHub Flow strategy for any new developments.

A quick summary for working with Github Flow:

1. For each change to the project create a new branch:
    "feat/mynewfeature"
2. Work on feature / changes on the branch
3. Push the changed branch to Github
4. Create new pull request
    From feat/-Branch
    To main-Branch
5. Have the changes reviewed by a maintainer
6. Fix any conflicts (automatically or with the contributing developers where necessary)
7. Merge changes to main in the Github pull request
